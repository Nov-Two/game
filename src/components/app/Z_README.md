# app 目录

## 概述

`app` 目录用于放置与**当前活动强相关**的应用级组件。这些组件通常与活动的业务逻辑、配置数据和特定功能紧密相关。

## 目录定位

该目录中的组件与以下目录的组件有所区别：

- **`ui/`** - 通用 UI 组件，可在任何场景复用
- **`module/`** - 模块级组件，用于特定功能模块
- **`app/`** - 活动级组件，与当前活动的配置和业务逻辑强相关

## 组件特点

放置在此目录的组件通常具有以下特征：

1. **依赖活动配置**：使用 `store.state.eventConfig` 中的活动配置数据
2. **活动业务逻辑**：实现活动特定的业务功能
3. **应用级使用**：在应用根组件或活动主页面中使用
4. **活动相关功能**：如活动规则、活动音效、活动头部等

## 当前组件

### AppHeader

应用头部组件，包含：

- 音效控制（使用活动配置的音效）
- 活动规则按钮（显示 `eventConfig.rule`）
- 玩家 UID 显示
- 活动标题插槽

**使用示例**：

```vue
<template>
  <AppHeader>
    <GfrHeading>活动标题</GfrHeading>
  </AppHeader>
</template>

<script setup>
import AppHeader from '@/components/app/header.vue'
</script>
```

## 使用规范

### 何时将组件放在此目录

✅ **应该放在这里**：

- 组件依赖 `eventConfig` 等活动配置
- 组件实现活动特定的业务逻辑
- 组件在应用级别使用，与活动生命周期相关
- 组件包含活动相关的功能（如活动规则、活动音效等）

❌ **不应该放在这里**：

- 通用的 UI 组件（应放在 `ui/`）
- 可复用的业务组件（应放在 `module/` 或其他业务目录）
- 与活动配置无关的组件

### 命名规范

- 组件文件名使用 kebab-case（如 `header.vue`）
- 组件名使用 PascalCase（如 `AppHeader`）
- 组件名建议以 `App` 前缀开头，便于识别

## 注意事项

1. **活动配置依赖**：这些组件通常依赖 `store.state.eventConfig`，确保在使用前已初始化配置
2. **活动特定性**：这些组件可能在不同活动间需要调整，注意保持灵活性
3. **状态管理**：活动相关的状态应通过 store 管理，组件通过 `storeToRefs` 访问

## 扩展建议

当需要添加新的活动相关组件时：

1. 确认组件是否与活动配置强相关
2. 遵循命名规范
3. 在组件中合理使用 store 中的活动配置
4. 保持组件的可维护性和可扩展性
